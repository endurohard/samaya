# Фронтенд: визуальный аудит и план оптимизации

Дата: 2026-06-06 · Объём: `services/frontend/src` (≈16 200 строк: `style.css` 4189, `app.js` 7357, 6 отдельных HTML-экранов).

## TL;DR

Визуально продукт уже близок к референсу (DIKIDI-style админка + чистая клиентская запись), но держится на **дублирующихся, рассинхронизированных стилях**. Главная проблема — нет единого источника правды: токены переопределяются в каждом HTML-файле, шкалы (размеры, отступы, радиусы) разъехались, бренд-цвет различался между админкой и клиентскими экранами. Это не «красиво/некрасиво», а **системная консистентность** — то, что ломает ощущение качества на стыках экранов.

Часть багов уже исправлена в этом проходе (см. раздел «Сделано»).

---

## Что нашли

### 1. Брендовый цвет различался по экранам ❗
- Админка (`style.css`): `--primary: #6366f1` (индиго).
- Клиентская запись (`book.html`), портал (`portal.html`): `--primary: #7c3aed` (фиолет).

Пользователь видит салон в двух разных «фирменных» цветах. Для бьюти-бренда фиолет уместнее, поэтому админка приведена к фиолету.

### 2. Токены живут в 6 местах
`book.html`, `portal.html`, `upload.html` и др. имеют **собственные `:root`** (book — 27 переменных, portal — 13). Любая правка палитры требует ручного синхрона по всем файлам — гарантированный дрейф.

### 3. Баг: `var(--radius)` без определения
В `style.css` дважды использовался `border-radius: var(--radius)`, но токен `--radius` **не был объявлен** → кнопки в финансовых модалках получали квадратные углы (0px). Исправлено.

### 4. Дублирующиеся конфликтующие правила кнопок
`.btn-success` и `.btn-danger` определялись **дважды** с разными цветами и padding:
- `.btn-danger`: вверху — прозрачный аутлайн, ниже — заливка `#dc2626`. Побеждала заливка → первая декларация была мёртвой.
- `.btn-success`: `var(--success)` (#10b981) против хардкода `#16a34a`.

Исправлено: оставлены каноничные определения, дубли удалены.

### 5. Разъехавшиеся шкалы
| Что | Сколько разных значений | Норма |
|-----|------------------------|-------|
| `font-size` | **17** (9–56px, вкл. нечётные 15/17px) | 6–7 ступеней |
| `border-radius` | **11** (2,3,4,5,6,7,8,10,12,20,999) | 3–4 ступени |
| `padding/margin/gap` | ~20 значений, много вне 8pt-сетки (5,7,9,14,18,22) | сетка 4/8 |
| `box-shadow` | **42 inline-варианта** | 3 уровня |
| breakpoints | 540/600/640/700/800/900/1100 | 3–4 точки |

### 6. Шрифтовой стек не один
Три разных стека по файлам: `SF Pro Text/Inter`, `Segoe UI/Roboto`, `Segoe UI`. На разных ОС экраны выглядят по-разному.

### 7. Хардкод цвета
200+ hex-значений мимо токенов (`#f1f5f9`, `#dc2626`, `#16a34a`…). Состояния (success/danger/warn) каждый раз пересобираются вручную → невозможно перекрасить тему централизованно.

## Сделано в этом проходе (в `style.css`)

1. **Единый бренд-фиолет** `--primary: #7c3aed` + согласованные `--primary-hover`/`--primary-soft` и активные состояния сайдбара. Теперь админка совпадает с book/portal.
2. **Добавлен слой дизайн-токенов** в `:root`: радиусы (`--radius-sm/--radius/--radius-lg/--radius-pill`), тени (`--shadow-sm/md/lg`), типошкала (`--fs-xs…--fs-2xl`), шкала отступов 8pt (`--space-1…--space-6`), `--font-sans`.
3. **Исправлен баг `--radius`** — больше нет квадратных кнопок в модалках.
4. **Удалены дубли** `.btn-success`/`.btn-danger`; `.btn-danger` приведён к единой заливке через `--danger`/`--danger-hover`.

Сборка цела: скобки в CSS сбалансированы (1046/1046), висячих `var(--radius)` не осталось.

## План оптимизации (по приоритету)

### P0 — консистентность ✅ СДЕЛАНО
1. ✅ **Токены вынесены в `tokens.css`** и подключены во все app-экраны (index/book/portal/upload/review). Виджет оставлен самодостаточным (namespace `--w-*`). Добавлен в `build.mjs` (хешируется, ссылки переписываются).
2. **Хардкод-цвета через токены** — частично; массовую замену 200+ hex оставить на отдельный проход.
3. ✅ **Единый `--font-sans`** на `body` всех файлов (было 3 стека).

### P1 — шкалы ✅ ЧАСТИЧНО СДЕЛАНО
4. ✅ `font-size` сведён к шкале `--fs-xs…--fs-3xl` (8 ролей). Доминанты (11/12/13/14/18/22) сохранены пиксель-в-пиксель; выбросы 9/10/15/17/20/24 свёрнуты к ближайшей ступени. Display-размеры (40/42/56) и iOS-защита `16px !important` оставлены сырыми намеренно. Итого 274 использования токенов.
5. ✅ `border-radius` сведён к `--radius-sm/md/lg/pill`. Доминанты 6/8/10 сохранены точно; 2/3/4/5/7→sm, 12→lg, 20→pill. 118 использований.
6. ⬜ Перевести отступы на 8pt-сетку (`--space-*`) — остаётся (риск: меняет плотность вёрстки, нужна визуальная проверка).
7. ⬜ Зафиксировать 3 breakpoint'а: `640 / 900 / 1200` — остаётся (риск: меняет точки перекомпоновки).
8. ✅ `box-shadow` — elevation-тени сведены к `--shadow-sm/md/lg` (значения по реальным доминантам, вид сохранён). Функциональные свечения (статус-точка, оверлей) оставлены. 29 использований токенов.

### P2 — качество ✅ ЧАСТИЧНО СДЕЛАНО
9. ✅ **Единый `:focus-visible`** — кольцо фокуса `--ring` для всех интерактивных элементов при навигации с клавиатуры (через `:where()` с нулевой специфичностью, мышь не затрагивается).
10. ✅ **Бренд-чистка**: все индиго-остатки старого бренда (`#6366f1` / `rgba(99,102,241)`) убраны из CSS **и** JS — focus-кольца, tap-highlight, `theme-color`, дефолтные цвета услуг, палитра графиков, статус «completed». Источников индиго в `src/` больше нет.
11. ⬜ Тёмная тема через `@media (prefers-color-scheme: dark)` — теперь дёшево после токенов, но требует dark-значений для остатков хардкод-цветов.
12. ⬜ Проверить контраст `--text-muted` на мелком тексте (WCAG AA).

### P2 — качество интерфейса
8. **Состояния компонентов**: единый `:focus-visible` (контур `--primary` + offset) для доступности с клавиатуры — сейчас фокус-стили разрозненны.
9. **Микровзаимодействия**: единые `transition` (150ms ease) и hover/active через токены теней.
10. **Тёмная тема**: с токенами добавляется `@media (prefers-color-scheme: dark)` переопределением `:root` — почти бесплатно после P0.
11. **Контраст**: проверить `--text-muted` (#9aa3b2) на белом — на мелком тексте близко к границе WCAG AA.
