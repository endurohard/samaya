# DIKIDI Finance — Деталь

Углублённый разбор модуля **Финансы** для реализации в samaya. Дополняет верхнеуровневый [[dikidi-feature-map#Финансы]]. Используется как референс при проектировании `finance-service`.

Ссылки: [[dikidi-screenshots-index]] (где какие скриншоты), [[../services/booking-service]] (источник доходов через продажи), [[dikidi-feature-map]].

## Структура раздела

Левое подменю «Финансы» раскрывается в **5 sub-tab**:

1. **Доходы и расходы** — главный экран движения денег
2. **Счета и кассы** — справочник + остатки
3. **Эквайринг** — настройка POS-терминалов
4. **Контрагенты** — справочник поставщиков/прочих
5. **Кассовые операции** — журнал 54-ФЗ операций онлайн-кассы

## 1. Доходы и расходы

Главный sub-tab. Скриншоты: `001_finance_cashflow.png` (день), `006_…_month.png`, `007_…_year.png`.

### Период-фильтр (top bar)

Pill-переключатель: **День / Неделя / Месяц / Год** + диапазонный picker слева. По умолчанию — день. Влияет на KPI-плитку, donut и группировку строк журнала.

### KPI-плитка (4 показателя)

| Метрика | Описание |
|---|---|
| Остаток на начало периода | Сумма по всем счетам на 00:00 первого дня |
| Доходы | Сумма всех income-операций за период |
| Расходы | Сумма всех expense-операций за период |
| Остаток на конец периода | Доходы − Расходы + Остаток на начало |

Каждая плитка с donut'ом, разбивающим на `Наличные / Безналичные` (по типу счёта-источника операции).

### Donut по статьям расходов

Видны статьи Samaya: `возврат клиентке`, `аптека`, `вб` (валберис), `Чистовье`, `зп наре`, `цветы`, `еда`, `кофе`, `мусор`, `сослан`, `Расходный материал`, `Вода`, `телефон на счет`, `такси`. Статьи кастомные per-company, заводятся owner'ом.

### Журнал операций (нижняя таблица)

Колонки:
- **Тип** — стрелка вверх/вниз (in/out) или иконка перемещения
- **Счёт/Касса** — источник для расхода, целевой для дохода
- **Получатель/Плательщик** — контрагент или текст
- **Статья движения** — категория из справочника
- **Сумма** — отрицательная для расхода, в валюте счёта
- **Примечание** — свободный текст
- **Создано** — дата + автор (user)

Строки кликабельны → открывают модалку редактирования.

### Action-bar (кнопки сверху)

| Кнопка | Открывает | Скриншот |
|---|---|---|
| `+ Доход` | Модалка дохода | `002_finance_modal_income.png` |
| `+ Расход` | Модалка расхода | `003_finance_modal_expense.png` |
| `Перемещение` | Модалка transfer между счетами | `004_finance_modal_transfer.png` |
| `Изменение остатков` | Модалка корректировки | `005_finance_modal_balance_change.png` |
| `Фильтры` | Drawer фильтров (статья, контрагент, счёт, автор) | — |
| `Печать` | Печатная форма реестра | — |

### Модалки операций — поля

**+ Доход** (`002`):
- Счёт/Касса (dropdown из справочника)
- Статья движения (dropdown «Доходы» категорий)
- Контрагент (опционально)
- Сумма (NUMERIC)
- Дата операции (по умолчанию сегодня)
- Примечание

**+ Расход** (`003`):
- Те же поля + статья из «Расходов»

**Перемещение** (`004`):
- Со счёта (dropdown)
- На счёт (dropdown)
- Сумма
- Дата
- Примечание
- При сохранении — создаётся **2 операции** (расход на источнике + доход на цели), связанные одним `transfer_id`.

**Изменение баланса** (`005`):
- Счёт
- Новый остаток (или дельта)
- Причина (свободный текст)
- Дата
- Используется для корректировки расхождений с фактом (инвентаризация кассы).

## 2. Счета и кассы

Скриншот: `008_finance_accounts.png`. Реальные данные Samaya на момент снятия:

| Название | Тип | Текущий остаток | Ответственный |
|---|---|---|---|
| Касса | Касса | 0.45 ₽ | (owner) |
| Расчётный счёт | Расчётный счёт | 38 094 659.60 ₽ | (owner) |
| Зухра | Личный счёт | 203 798 719.90 ₽ | (owner) |

То есть в DIKIDI типы счетов: `Касса`, `Расчётный счёт`, плюс именованные кастомные (личные счета владельца). Все участвуют в общем cashflow одинаково.

### Карточка счёта (`011_finance_account_detail.png`)

При клике — открывается отдельная страница с историей операций именно по этому счёту, фильтрами по периоду и кнопкой `Изменить`.

### Модалки

**+ Касса** (`009_finance_modal_add_cashbox.png`):
- Название
- Ответственный (user)
- Начальный остаток
- Активность (toggle)

**+ Счёт** (`010_finance_modal_add_account.png`):
- Те же поля + Тип (Расчётный/Личный/Прочее)
- Реквизиты (банк, БИК, рассчётный — опционально)

## 3. Эквайринг

Скриншот: `012_finance_acquiring.png`. Настройка интеграций с POS-терминалами банков. Поля per-терминал: банк, ID терминала, комиссия, счёт зачисления (привязка к счёту из (2)).

Для samaya MVP — отложено; можно делать ручной ввод операций с категорией «эквайринг».

## 4. Контрагенты

Скриншоты: `013_finance_counterparties.png`, `014_finance_counterparty_detail.png`, `018_09_finance_Контрагенты_modal.png`.

Таблица справочника:

| Колонка | Описание |
|---|---|
| Имя/Название | Юр.лицо или ФИО |
| ИНН | Налоговый ID |
| Тип | Поставщик / Покупатель / Прочее |
| Активность | toggle |

Карточка контрагента (`014`) — список всех cashflow-операций с этим контрагентом + сальдо.

Модалка «+ Контрагент» (`018_…_modal`):
- Имя/Название
- ИНН
- Телефон / Email (опционально)
- Тип
- Примечание

## 5. Кассовые операции (54-ФЗ)

Скриншот: `019_09_finance_Кассовые_операции.png`. Журнал операций онлайн-кассы (фискальный регистратор по 54-ФЗ).

Колонки:
- Дата/время
- ID документа
- Документ (чек, возврат)
- Тип операции (приход / возврат прихода / расход)
- Статус (отправлен в ОФД / ошибка)
- Кассир (user)
- Сумма

Кнопка `Настройка оборудования` — настройка фискальника.

Для samaya MVP — отложено (Phase 1+); пока работаем без 54-ФЗ.

## Связи с другими модулями

| Источник денег | Куда попадает |
|---|---|
| Оплата записи в [[../services/booking-service]] | Доход на счёт-источник кассы |
| Продажа в Продажах | То же + позиции в кассовый чек |
| Закупка в Товарах | Расход на счёт-источник |
| Зарплата (выплата) | Расход + операция «зп <имя>» |

## Что нужно для samaya finance-service

Минимум для MVP (Phase 1):
1. Таблицы: `accounts` (счета), `cashflow_categories` (статьи), `counterparties`, `cashflow_operations` (income/expense/transfer/adjust), `transfer_groups` (для парных операций перемещения).
2. KPI-расчёт: 4 числа за период через 1 SQL c GROUP BY type.
3. Модалки 5 типов: income / expense / transfer / balance_change / + 2 справочники (account, counterparty).
4. Период-pills (день/неделя/месяц/год) — переиспользовать паттерн из [[../services/frontend-service]] журнала.
5. **НЕ делать в MVP**: эквайринг, 54-ФЗ, печатные формы, donut по статьям (только список).

Следующий ADR — `decisions/2026-XX-XX-finance-service-mvp.md` зафиксирует выбранный scope.
