## Журнал записей

Главный экран. День/неделя view, колонки = мастера, строки = время (09:00–20:00 шаг 30 мин).

**Карточка записи содержит:**
- Время начала
- Имя клиента + телефон
- Список услуг (например, "ЛЛ живота", "ЛЛ рук")
- Примечание мастера ("Пери Живот 200т Лобок 50т Руки 200т")
- Бейдж `NEW` (новая запись)
- Статус подтверждения (зелёная галочка)
- Индикатор сообщений

**Действия:** `+ Добавить запись`, `Оформить продажу` (превращает запись в продажу/чек).

**Боковой виджет:** мини-календарь месяца + "Финансы за сегодня" с разбивкой Наличные / Безналичные.

Связи: [[#График работы]] (источник свободных слотов), [[#Клиенты]] (карточка клиента), [[#Продажи]] (запись → продажа), [[#Финансы]] (касса дня).

## График работы

Сетка: строки = мастера (с аватарами), колонки = дни месяца (видно ~17 дней горизонтально).

В ячейке: рабочий интервал `10:00 / 20:00` (типовой), либо нестандартный `09:00 / 22:00`. Пустая ячейка = выходной.

**Боковой виджет:** "Выделите в графике один или несколько дней для настройки расписания" — bulk-edit.
**Toggle:** "Отобразить пометки о записях в графике".
**Действие:** `+ Добавить сотрудника`.

Это **мастер-данные доступности**. Booking-сервис читает отсюда → вычисляет свободные слоты с учётом длительности услуги.

## Клиенты

Подразделы:
- **Список клиентов** — таблица (Клиент, Визиты, Последний визит, Скидка, Средний чек, Потрачено). 6963 клиента в Samaya. У клиента — телефон, аватар, email-флажок.
- **Возвращаемость** — аналитический отчёт: Клиентов за период / Вернувшиеся / Потерянные, в разрезе мастеров. Метрики: % новых, % вернувшихся, % потерянных.
- **Бонусная программа** — начисление и списание бонусов, скидки.
- **Рассылка** — массовые сообщения. Способы: **WhatsApp** (видно), вероятно SMS. Поля: название, получателей (фильтр по сегментам), тип, статус (`Отправлено`), время создания.
- **Звонки** — лог звонков (телефония, аналог [[../concepts/telephony]] из food-flow).
- **Сертификаты** — продажа/использование подарочных.
- **Чаевые** — учёт чаевых мастерам.
- **Отзывы** — сбор и модерация.
- **Штрафы** — нарушения условий записи (no-show, опоздания).

**Сегменты (правый сайдбар):**
- `Постоянные` — 2+ записи за 3 месяца.
- `Спящие` — не записывались более 3 месяцев.
- `Пропавшие` — не записывались более 6 месяцев.
- `Не посещали` — нет записей вообще.
- `Новые` — за выбранный период (например, 19.04–25.04).
- `Заблокированы` — не могут записаться онлайн.
- `Удалены`, `Штрафники`.

Это **классическая RFM-сегментация**, посчитанная по `bookings`.

## Зарплата

Подразделы:
- **Расчет зарплаты** — за период (Сегодня/Вчера/Неделя/Месяц). Колонки: `% с продаж`, `% с продаж салона`, `Оклад/Повременная`, `Гарантированная`, **К начислению**. У сотрудников разные роли: Врач-косметолог, менеджер по продажам, асистент, админ.
- **Взаиморасчеты** — кто кому должен.
- **Начисления** — журнал начислений.
- **Схемы зарплаты** — конфигурация. Видны типы:
  - `Ставка` (фикс. оклад)
  - `Ставка + % с продаж (- скидка)`
  - `% с продаж`

То есть схема зарплаты — **per-employee, с датой начала действия**, можно менять во времени (история ставок). У одного сотрудника видно несколько схем с разными датами (versioning).

## Продажи

Не открыта вкладка, но из главного экрана видно кнопку `Оформить продажу`. Это:
- Чек = услуги мастера + проданные товары.
- Скидки, оплата (наличные/безнал), эквайринг.
- Связь: запись → продажа (один-ко-многим? или один-к-одному?).

## Финансы

Подразделы:
- **Доходы и Расходы** — главный финансовый экран:
  - Остаток на начало периода / Доходы / Расходы / Остаток на конец, с donut-чартами.
  - Разбивка остатка: `Наличные` / `Безналичные` (для Samaya: 202 млн / 38 млн ≈ 240 млн остаток — реальные обороты косметологии).
  - Расход по статьям: `возврат клиентке`, `аптека`, `вб` (валберис?), `Чистовье` (бренд расходников), `зп наре`, `цветы`, `еда`, `кофе`, `мусор`, `сослан`, `Расходный материал`, `Вода`, `телефон на счет`, `такси`.
  - Действия: `Доход`, `Расход`, `Перемещение` (между счетами), `Фильтры`, `Печать`, `Изменение остатков`.
  - **Журнал операций**: Тип (стрелка in/out) | Счет/Касса | Получатель/Плательщик | Статья движения | Сумма | Примечание | Создано (дата + автор).
- **Счета и кассы** — типы: `Касса`, `Расчетный счет`, плюс именованные (`Зухра` — личный счёт владельца). Поля: Текущий остаток, Ответственный, Активность.
- **Эквайринг** — настройка POS-терминалов и выгрузка.
- **Контрагенты** — поставщики/прочие. Поля: Имя/название, ИНН.
- **Кассовые операции** — операции онлайн-кассы (54-ФЗ). Поля: Дата, ID документа, Документ, Тип операции, Статус, Кассир, Сумма. Кнопка `Настройка оборудования` (фискальник).

Это полноценная **двойная бухгалтерия в упрощённом виде** + 54-ФЗ интеграция.

## Техническая реализация DIKIDI (извлечено 2026-04-26)

Подробнее — [[dikidi-api]].

**Стек:**
- jQuery 1.11.1 + кастомный фреймворк `sw` (НЕ React/Vue/Angular, чистый MPA)
- Bootstrap 3 custom themed
- Riot.js тэги (использованы в форме записи: `journal-record-master`, `journal-record-service-item`)
- PushStream.js — WebSocket real-time (сервер: `query.dikidi.net`)
- Server-side rendered HTML, AJAX фрагменты через `sw.result()` → `sw.apply()`
- Без webpack/vite — простая конкатенация JS/CSS с version-querystring

**Домены:**
- `beauty.dikidi.net` — публичный сайт + точка входа (логин-модал)
- `dikidi.net/ru/owner/` — owner-панель (весь бизнес-функционал)
- `auth.dikidi.net` — authentication API
- `chat.dikidi.net` — chat API
- `query.dikidi.net` — PushStream WebSocket
- `f1.dikidi.net`, `f2.dikidi.net` — CDN файлы/фото
